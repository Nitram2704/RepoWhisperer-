**`file_path_to_module_name(file_path: str, root: str) -> str`:**
Convert a file path to a dotted Python module name anchored at `root`.
```python
def file_path_to_module_name(file_path: str, root: str) -> str:
    """Convert file path to dotted module name.

    Examples:
        src/docgen/parser/__init__.py, root=src → docgen.parser
        src/docgen/store.py, root=src            → docgen.store
        src/docgen/llm/groq.py, root=src         → docgen.llm.groq
    """
    p = Path(file_path).resolve()
    r = Path(root).resolve()
    relative = p.relative_to(r)           # docgen/parser/__init__.py
    no_suffix = relative.with_suffix('')  # docgen/parser/__init__
    if no_suffix.name == '__init__':
        no_suffix = no_suffix.parent      # docgen/parser
    return '.'.join(no_suffix.parts)      # 'docgen.parser'
```

Edge cases to handle:
- If `file_path` is not relative to `root`, catch `ValueError` from `relative_to()` and fall back to using the filename without extension as the module name
- Non-Python files (JS/TS): `with_suffix('')` removes `.ts`/`.js` correctly
- Empty result (e.g., `__init__.py` at root): return the parent directory name instead of empty string

**`group_chunks_by_module(chunks: list[CodeChunk], root: str) -> dict[str, list[CodeChunk]]`:**
```python
def group_chunks_by_module(chunks: list[CodeChunk], root: str) -> dict[str, list[CodeChunk]]:
    """Group CodeChunks by their source file, keyed by dotted module name."""
    groups: dict[str, list[CodeChunk]] = defaultdict(list)
    for chunk in chunks:
        module = file_path_to_module_name(chunk.file_path, root)
        groups[module].append(chunk)
    return dict(groups)
```

This is critical for performance: one LLM call per module (file), NOT per chunk. A 50-file project with 200 functions = 50 LLM calls, not 200.

**`validate_output_dir(output_dir: Path, source_dir: Path) -> None`:**
Raises `ValueError` if the output would be destructive.
```python
def validate_output_dir(output_dir: Path, source_dir: Path) -> None:
    """Prevent accidental overwrites of source files."""
    out = output_dir.resolve()
    src = source_dir.resolve()
    if out == src:
        raise ValueError(
            f"Output directory {out} is the same as source directory. "
            "Use --output-dir to specify a safe output location (e.g., docs/)."
        )
```

Note: We intentionally allow `output_dir` to be a subdirectory of `source_dir` (like `./docs`), because that's the default and expected use case. We only block `output_dir == source_dir` to prevent the tool from overwriting `README.md` at the project root.

**`safe_write(output_dir: Path, relative_path: str, content: str) -> Path`:**
Writes content to `output_dir / relative_path` with traversal protection.
```python
def safe_write(output_dir: Path, relative_path: str, content: str) -> Path:
    """Write content safely, preventing path traversal."""
    out_file = (output_dir / relative_path).resolve()
    if not out_file.is_relative_to(output_dir.resolve()):
        raise ValueError(f"Path traversal detected: {relative_path}")
    out_file.parent.mkdir(parents=True, exist_ok=True)
    out_file.write_text(content, encoding="utf-8")
    return out_file
```

**`write_docs(readme_md: str, module_docs: dict[str, str], output_dir: Path, source_dir: Path) -> list[Path]`:**
Top-level function that writes all generated docs to disk.
```python
def write_docs(readme_md: str, module_docs: dict[str, str], output_dir: Path, source_dir: Path) -> list[Path]:
    """Write README and per-module API docs. Returns written paths."""
    validate_output_dir(output_dir, source_dir)
    written = []

    # Write README.md
    written.append(safe_write(output_dir, "README.md", readme_md))

    # Write per-module API docs: docs/api/docgen.parser.md (flat names)
    for module_name, content in module_docs.items():
        filename = f"{module_name}.md"
        written.append(safe_write(output_dir / "api", filename, content))

    return written
```

Module name → filename uses flat dotted names (`docs/api/docgen.parser.md`) — simpler for v1 than nested directories.

Add the following method to the existing `VectorRepository` class in `src/docgen/store.py`:

```python
def list_files(self) -> list[str]:
    """Return unique file_path values stored in the collection.

    Used by `docgen generate` to reconstruct the module list
    without re-parsing source files.
    """
    if self.count() == 0:
        return []
    results = self._collection.get(include=["metadatas"])
    seen: set[str] = set()
    files: list[str] = []
    for meta in results.get("metadatas", []):
        fp = meta.get("file_path", "")
        if fp and fp not in seen:
            seen.add(fp)
            files.append(fp)
    return files
```

This is necessary because `docgen generate` does NOT re-parse source files — it must reconstruct the module list from whatever was previously indexed. Without this method, the `generate` command would either need the user to re-specify every file or would crash.

**Design note:** We use an ordered de-duplication (seen set + append) rather than just `set()` to maintain a deterministic file order across runs, which helps with reproducible output.

**`async generate_all_docs(module_groups: dict[str, list[CodeChunk]], repo: VectorRepository, provider, provider_name: str = "gemini", on_progress: callable | None = None) -> tuple[str, dict[str, str]]`:**

Returns `(readme_text, module_docs_dict)`.

```python
async def generate_all_docs(
    module_groups: dict[str, list[CodeChunk]],
    repo: VectorRepository,
    provider,
    provider_name: str = "gemini",
    on_progress = None,
) -> tuple[str, dict[str, str]]:
    """Generate README + per-module docs concurrently, throttled by RPM."""
    rpm = RPM_BY_PROVIDER.get(provider_name, 10)
    limiter = AsyncLimiter(max_rate=rpm, time_period=60)

    # Generate README first (uses all chunks as context)
    async with limiter:
        readme_text = await asyncio.to_thread(
            generate_docs,
            "Generate a README.md summarizing this project: its purpose, "
            "main modules, installation, and usage examples.",
            repo,
            provider,
        )
    if on_progress:
        on_progress()

    # Generate per-module docs concurrently
    module_docs: dict[str, str] = {}

    async def generate_one(module_name: str) -> tuple[str, str]:
        query = (
            f"Generate API reference documentation for module '{module_name}'. "
            "Document all public functions and classes with their signatures, "
            "parameters, return types, and descriptions."
        )
        async with limiter:
            text = await asyncio.to_thread(
                generate_docs, query, repo, provider
            )
        if on_progress:
            on_progress()
        return module_name, text

    tasks = [generate_one(name) for name in module_groups]
    results = await asyncio.gather(*tasks)
    for name, text in results:
        module_docs[name] = text

    return readme_text, module_docs
```

**Key design decisions:**
- `asyncio.to_thread()` bridges the synchronous `provider.generate()` to async without rewriting existing providers — verified correct in 05-RESEARCH.md
- `aiolimiter.AsyncLimiter` uses leaky-bucket: first N calls burst instantly, subsequent calls wait for capacity. This maximizes throughput within rate limits.
- README is generated first (separate call) to ensure it always has context from the entire index, not just one module.
- `on_progress` callback is called after each successful generation — Plan 02 will connect this to `rich.Progress.advance()`.
