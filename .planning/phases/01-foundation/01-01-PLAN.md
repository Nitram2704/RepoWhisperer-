<task type="auto">
  <name>Task 2: Create config.py, ui.py, and models.py modules</name>
  <files>src/docgen/config.py, src/docgen/ui.py, src/docgen/models.py</files>
  <action>
Create three modules inside `src/docgen/`:

**config.py** — Secure API key loading:
- Import `os`, `typer`, `load_dotenv` from dotenv, `Console` from rich.console
- Create `_error = Console(stderr=True)` for styled error output
- Implement `load_config() -> dict` that:
  1. Calls `load_dotenv()` (no-op if .env absent)
  2. Reads `os.getenv("DOCGEN_API_KEY")`
  3. If key is missing/empty: prints styled error to stderr explaining the variable name and how to set it, then raises `typer.Exit(code=1)`
  4. Returns `{"api_key": api_key}`
- NEVER print or log the API key value. Only reference the variable name in error messages.

**ui.py** — Rich spinner helper:
- Import `contextmanager` from contextlib, `Progress`, `SpinnerColumn`, `TextColumn` from rich.progress
- Implement `spinner(description: str)` as a `@contextmanager` that:
  1. Creates `Progress(SpinnerColumn(), TextColumn("[progress.description]{task.description}"), transient=True)`
  2. Adds a task with `total=None` (indeterminate)
  3. Yields (caller does work inside the `with` block)

**models.py** — Data contract:
- Import `dataclass` from dataclasses, `Optional` from typing
- Define `@dataclass` class `CodeChunk` with fields:
  - `file_path: str` — absolute path to source file
  - `language: str` — "python", "javascript", or "typescript"
  - `chunk_type: str` — "function", "class", "module", or "block"
  - `name: str` — identifier (function/class name or module name)
  - `content: str` — raw source code of the chunk
  - `start_line: int` — first line number in the source file
  - `end_line: int` — last line number in the source file
  - `docstring: Optional[str] = None` — extracted docstring if present
  - `parent: Optional[str] = None` — parent class/module name if nested

This dataclass is the contract between the parser (Phase 2) and all downstream stages. Do not add methods beyond `__post_init__` if needed for validation.
  </action>
