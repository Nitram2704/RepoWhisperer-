<task type="auto">
  <name>Task 1: Create main.py with three CLI commands wired to config and spinner</name>
  <files>src/docgen/main.py</files>
  <action>
Create `src/docgen/main.py` with:

1. Import `typer`, `from docgen.config import load_config`, `from docgen.ui import spinner`
2. Create `app = typer.Typer(help="Generate documentation for your codebase.", pretty_exceptions_show_locals=False)`
3. Three `@app.command()` functions:

**run(path: str = typer.Argument(..., help="Path to source directory")):**
- Docstring: "Scan, embed, and generate docs in one step."
- Calls `cfg = load_config()` (will fail-fast if API key missing)
- Wraps body in `with spinner("Running full pipeline..."):`
- Body is a stub: `pass` (or a brief `typer.echo("Pipeline complete.")` after spinner exits)

**index(path: str = typer.Argument(..., help="Path to source directory")):**
- Docstring: "Build the vector index without generating docs."
- Does NOT call `load_config()` (indexing is local-only, no API key needed)
- Wraps body in `with spinner("Indexing..."):`
- Body is a stub

**generate():**
- Docstring: "Generate docs from an existing index."
- Calls `cfg = load_config()` (generation needs LLM API key)
- Wraps body in `with spinner("Generating documentation..."):`
- Body is a stub

4. Add `if __name__ == "__main__": app()` at the bottom.

Key constraints from research:
- `pretty_exceptions_show_locals=False` — never expose secrets in tracebacks
- Use `typer.Argument(...)` (with `...` as default) to make path required
- Three flat commands on one `Typer()` instance — do NOT use `app.add_typer()`
  </action>
  <verify>
- `uv run python -m docgen.main --help` shows run, index, generate commands
  </verify>
  <done>main.py exists with three commands, imports config and ui, pretty_exceptions_show_locals is False</done>
