3. Create `src/docgen/embedder.py`:

**Imports:**
- `from fastembed import TextEmbedding`
- `from rich.console import Console`

**class Embedder:**

`__init__(self, model_name: str = "BAAI/bge-small-en-v1.5"):`
- Store model_name
- Print a one-time notice to stderr if the model will need downloading:
  ```python
  console = Console(stderr=True)
  console.print(f"[dim]Loading embedding model: {model_name} (first run may download ~50MB)...[/dim]")
  ```
- Initialize: `self._model = TextEmbedding(model_name=model_name)`
- The model download happens lazily on first embed call, but TextEmbedding constructor may also trigger it. The notice covers both cases.

`embed(self, texts: list[str], batch_size: int = 32) -> list[list[float]]:`
- Call `self._model.embed(texts, batch_size=batch_size)` — returns a generator
- Convert to list: `list(self._model.embed(texts, batch_size=batch_size))`
- Each embedding is a numpy array — convert to plain list: `[emb.tolist() for emb in raw_embeddings]`
- Return the list of float lists

`@property`
`dimension(self) -> int:`
- Return 384 (BAAI/bge-small-en-v1.5 produces 384-dimensional vectors)

**Critical design notes:**
- fastembed uses ONNX Runtime and runs entirely locally — NO network calls after initial model download
- The model is ~50MB and cached in the user's home directory (~/.cache/fastembed or platform equivalent)
- batch_size=32 balances memory usage and throughput for the 50-file target
- All embedding happens on CPU — no GPU required

**Part A — Create `src/docgen/ingest.py`:**

**Imports:**
- `pathlib.Path`
- `typer` (for echo/styled output)
- `filelock.Timeout`
- `from docgen.parser import parse_directory`
- `from docgen.embedder import Embedder`
- `from docgen.store import VectorRepository`
- `from docgen.ui import spinner`

**`run_ingest(repo_path: str, chroma_dir: str | None = None) -> dict:`**

This is the single entry point for the full ingest pipeline:

1. Resolve paths:
   - `repo = Path(repo_path).resolve()`
   - `chroma = chroma_dir or str(repo / ".chroma")`
   - Validate `repo.is_dir()` — if not, print error and raise `typer.Exit(code=1)`

2. Parse phase — with spinner("Parsing source files..."):
   - `chunks = parse_directory(repo)`
   - If no chunks: print "No parseable files found." and return `{"parsed": 0, "embedded": 0, "skipped": 0}`

3. Embed phase — with spinner(f"Embedding {len(chunks)} chunks..."):
   - `embedder = Embedder()`
   - `texts = [chunk.content for chunk in chunks]`
   - `embeddings = embedder.embed(texts, batch_size=32)`

4. Store phase — with spinner("Storing in vector database..."):
   - Wrap in try/except for `filelock.Timeout`:
     ```python
     try:
         store = VectorRepository(chroma_dir=chroma)
         result = store.upsert_chunks(chunks, embeddings)
     except Timeout:
         typer.echo("Error: Another docgen process is currently indexing. Please wait and try again.", err=True)
         raise typer.Exit(code=1)
     ```

5. Report results:
   - `typer.echo(f"Indexed {result['upserted']} chunks ({result['skipped']} unchanged, skipped).")`
   - Return `{"parsed": len(chunks), **result}`

**Part B — Update `src/docgen/main.py`:**

Update the existing `index` command (currently a stub) to call `run_ingest`:

1. Add import at top: `from docgen.ingest import run_ingest`
   - Use a lazy import inside the function body to avoid slow startup:
     ```python
     @app.command()
     def index(path: str = typer.Argument(..., help="Path to source directory")):
         """Build the vector index without generating docs."""
         from docgen.ingest import run_ingest

     ```

2. Remove the existing stub body (spinner + pass) from the index command. The spinners are now inside `run_ingest` itself.

3. Do NOT modify the `run` or `generate` commands — they remain stubs for now.

- The `run` command will eventually call `run_ingest` + generation, but that wiring belongs to Phase 5
- All embedding happens locally via fastembed ONNX — verify no network calls by checking that fastembed.TextEmbedding does not make HTTP requests after model is cached
