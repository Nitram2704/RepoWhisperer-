**Imports:**
- `hashlib`, `pathlib.Path`
- `chromadb` (PersistentClient)
- `filelock.FileLock`
- `docgen.models.CodeChunk`

**Hash function (module-level):**
```python
def _chunk_hash(chunk: CodeChunk) -> str:
    """SHA-256 of file_path + name + content for deterministic dedup."""
    payload = f"{chunk.file_path}::{chunk.name}::{chunk.content}"
    return hashlib.sha256(payload.encode("utf-8")).hexdigest()
```

**class VectorRepository:**

`__init__(self, chroma_dir: str = ".chroma"):`
- Store `self._chroma_dir = Path(chroma_dir)`
- Create `self._chroma_dir.mkdir(parents=True, exist_ok=True)`
- Create `self._lock = FileLock(str(self._chroma_dir / "docgen.lock"), timeout=5)`
  - Use timeout=5 seconds. If another process holds the lock, this raises `filelock.Timeout` which the caller can catch.
- Create `self._client = chromadb.PersistentClient(path=str(self._chroma_dir))`
- Create collection: `self._collection = self._client.get_or_create_collection(name="docgen")`
  - Do NOT pass an embedding_function here. Plan 02 will provide embeddings externally via the `embeddings` parameter on add/upsert. This avoids ChromaDB pulling in its own default embedding model.

`upsert_chunks(self, chunks: list[CodeChunk], embeddings: list[list[float]]) -> dict:`
- Acquires `self._lock` using `with self._lock:`
- For each chunk, compute `_chunk_hash(chunk)` as the id
- Check which ids already exist in the collection with matching hashes by calling `self._collection.get(ids=batch_ids, include=["metadatas"])`
- Compare existing metadata hashes against computed hashes. Skip chunks where hash matches (content unchanged).
- For chunks that are new or changed, call `self._collection.upsert(ids=..., documents=..., embeddings=..., metadatas=...)` where:
  - `ids` = list of hash strings
  - `documents` = list of `chunk.content` strings
  - `embeddings` = corresponding embeddings from the input list
  - `metadatas` = list of dicts with keys: `file_path`, `language`, `chunk_type`, `name`, `start_line`, `end_line`, `content_hash` (the SHA-256)
- Return a summary dict: `{"total": len(chunks), "skipped": N, "upserted": M}`

`query(self, text: str, n_results: int = 5, embedding: list[float] | None = None) -> dict:`
- If `embedding` is provided, use `self._collection.query(query_embeddings=[embedding], n_results=n_results)`
- If `embedding` is None, use `self._collection.query(query_texts=[text], n_results=n_results)` (fallback for testing)
- Return the raw ChromaDB result dict

`count(self) -> int:`
- Return `self._collection.count()`

`clear(self) -> None:`
- Acquires lock, then `self._client.delete_collection("docgen")` and recreates it
- Use `with self._lock:` for safety

**Important design notes:**
- The lock file is `docgen.lock` (not just `lock`) to avoid conflicts with ChromaDB's own internal files
- `upsert_chunks` processes in batches internally. ChromaDB's upsert handles batch sizes well, but if chunks > 500, split into batches of 100 to avoid memory issues
- All metadata values must be str, int, float, or bool — no None values. Use empty string `""` for optional fields that are None (e.g., docstring, parent)

    chunks = [

                  name='hello', content='def hello(): pass', start_line=1, end_line=1),
