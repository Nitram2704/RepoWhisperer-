**Module-level constants (instantiated ONCE, reused across all files):**
- `JS_LANGUAGE = Language(tsjs.language())` — capsule API, NOT integer
- `TS_LANGUAGE = Language(tsts.language_typescript())` — for .ts files
- `TSX_LANGUAGE = Language(tsts.language_tsx())` — for .tsx files (DIFFERENT grammar from TS)
- `_js_parser = Parser(JS_LANGUAGE)` — language passed to constructor, NOT set_language()
- `_ts_parser = Parser(TS_LANGUAGE)`
- `_tsx_parser = Parser(TSX_LANGUAGE)`

**Query string `_JS_TS_QUERY_SRC`:**
```
(function_declaration name: (identifier) @name) @chunk
(class_declaration name: (identifier) @name) @chunk
(method_definition name: (property_identifier) @name) @chunk
```
Note: Arrow functions assigned to variables (`const foo = () => {}`) lack a `name` field on the `arrow_function` node. For Phase 2, handle named functions and classes. Arrow functions without extractable names can be emitted as `<anonymous>` or skipped — but named function declarations and class declarations MUST produce chunks.

**`parse_js_ts(path: Path, source: str) -> list[CodeChunk]`:**
- Select parser/language/lang_name based on `path.suffix.lower()`:
  - `.tsx` -> `_tsx_parser, TSX_LANGUAGE, "typescript"`
  - `.ts` -> `_ts_parser, TS_LANGUAGE, "typescript"`
  - `.js` -> `_js_parser, JS_LANGUAGE, "javascript"`
- Parse: `tree = parser.parse(bytes(source, "utf8"))`
- Query: `query = language.query(_JS_TS_QUERY_SRC)`
- Captures: `captures = query.captures(tree.root_node)` — returns `dict[str, list[Node]]` (v0.22+ API, NOT tuple list)
- Iterate `captures.items()`, process only `capture_name == "chunk"`:
  - For each node: get name via `node.child_by_field_name("name")`, decode with `.text.decode("utf8")`
  - If no name node, use `"<anonymous>"`
  - Determine `chunk_type`: `"class"` if `node.type == "class_declaration"`, else `"function"`
  - Line numbers: `start_line = node.start_point[0] + 1` (tree-sitter is 0-indexed, CodeChunk is 1-indexed)
  - Content: `"\n".join(source_lines[start_line - 1 : end_line])`
  - `docstring=None` (JSDoc extraction is out of scope for Phase 2)
  - `parent=None` for top-level, or extract parent class name for method_definition nodes

Import CodeChunk from `docgen.models`. Import tree_sitter_javascript as tsjs, tree_sitter_typescript as tsts, Language and Parser from tree_sitter.

CRITICAL API WARNINGS (from research):
- Do NOT use `parser.set_language()` — pass language to `Parser()` constructor
- Do NOT iterate captures as `for node, name in captures` — it returns a dict, not tuple list
- Do NOT use `Language(integer)` — grammar.language() returns a capsule object
- Do NOT use TS grammar for .tsx files — TSX has its own separate grammar

  <action>
Update `src/docgen/parser/__init__.py` to export the public API:

**`PARSEABLE_EXTENSIONS`** — module-level set: `{".py", ".js", ".ts", ".tsx"}`

**`parse_file(path: Path, source: str) -> list[CodeChunk]`:**
- Dispatches based on `path.suffix`:
  - `.py` -> `parse_python(path, source)`
  - `.js`, `.ts`, `.tsx` -> `parse_js_ts(path, source)`
- Returns empty list for unsupported extensions (defensive, should not happen if caller checks)

**`parse_directory(repo_root: Path) -> list[CodeChunk]`:**
- Calls `build_gitignore_spec(repo_root)` to load .gitignore patterns
- Walks `repo_root.rglob("*")` to find all files
- For each file:
  1. Skip if not `path.is_file()`
  2. Skip if `path.suffix not in PARSEABLE_EXTENSIONS`
  3. Skip if `not should_parse(path, repo_root, spec)` — THIS IS THE SECURITY GATE. Must run BEFORE `path.read_text()`. A file that fails should_parse must NEVER have its content read.
  4. Read content: `source = path.read_text(encoding="utf-8", errors="replace")`
  5. Dispatch to `parse_file(path, source)`
  6. Extend the chunks list
- Returns all collected CodeChunks

Import from siblings:
- `from docgen.parser.filter import build_gitignore_spec, should_parse`
- `from docgen.parser.python_parser import parse_python`
- `from docgen.parser.js_parser import parse_js_ts`
- `from docgen.models import CodeChunk`

The critical ordering invariant: `should_parse()` is called BEFORE `path.read_text()`. This ensures sensitive file content never enters memory.
  </action>
