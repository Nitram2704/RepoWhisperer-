3. Create `src/docgen/parser/filter.py` with:

**SENSITIVE_PATTERNS** — a module-level list of glob patterns for files that must NEVER be parsed:
```
"**/.env", "**/.env.*", "**/*.key", "**/*.pem", "**/*.pfx", "**/*.p12",
"**/credentials*", "**/secrets*", "**/*_secret*", "**/config/production*", "**/*.keystore"
```

**`is_sensitive(path: Path) -> bool`** — returns True if the path matches ANY pattern in SENSITIVE_PATTERNS. Use `path.match(pattern)` for each pattern. This function checks path metadata ONLY — never reads file content.

**`build_gitignore_spec(repo_root: Path) -> pathspec.PathSpec`** — reads `repo_root / ".gitignore"` if it exists, returns a `PathSpec.from_lines("gitwildmatch", lines)`. If `.gitignore` does not exist, returns an empty PathSpec (matches nothing).

**`should_parse(path: Path, repo_root: Path, spec: pathspec.PathSpec) -> bool`** — the single entry point for all filtering. Returns True ONLY if the file is safe to parse. Order is critical:
  1. Check `is_sensitive(path)` FIRST — if True, return False immediately
  2. Compute `relative = path.relative_to(repo_root)`, call `spec.match_file(str(relative))` — if True (file is gitignored), return False
  3. Return True (safe to parse)

The ordering ensures sensitive files are blocked even if not in .gitignore. The `spec.match_file()` receives a RELATIVE path (not absolute) — pathspec requires this to match gitignore patterns correctly.

Do NOT read file content anywhere in this module. Filtering is path-based only.
  </action>
  <verify>

- `uv run python -c "from pathlib import Path; from docgen.parser.filter import is_sensitive; assert is_sensitive(Path('project/.env')); assert is_sensitive(Path('keys/server.key')); assert is_sensitive(Path('certs/ca.pem')); assert not is_sensitive(Path('src/main.py')); print('PASS')"` prints PASS

<task type="auto">
  <name>Task 2: Create python_parser.py with ast-based CodeChunk extraction</name>
  <files>src/docgen/parser/python_parser.py</files>
  <action>
Create `src/docgen/parser/python_parser.py` with:

**`_ChunkVisitor(ast.NodeVisitor)`** — internal visitor class:
- Constructor takes `source_lines: list[str]`, `file_path: str`, `parent: str | None = None`
- Has `self.chunks: list[CodeChunk] = []`
- Has `_extract_content(node) -> str` that slices `source_lines[node.lineno - 1 : node.end_lineno]` and joins with `"\n"`

- **`visit_FunctionDef(node)`:** Appends a CodeChunk with `chunk_type="function"`, `language="python"`, `name=node.name`, content from `_extract_content`, `start_line=node.lineno`, `end_line=node.end_lineno`, `docstring=ast.get_docstring(node)`, `parent=self.parent`. Then creates a NEW `_ChunkVisitor` with `parent=node.name` and calls `nested.generic_visit(node)` to find nested functions. Extends `self.chunks` with `nested.chunks`. This nesting approach is CRITICAL — without creating a new visitor and calling generic_visit, nested functions/methods inside classes will be silently skipped.

- **`visit_AsyncFunctionDef(node)`:** Delegates to `self.visit_FunctionDef(node)` — async functions produce identical chunks.

- **`visit_ClassDef(node)`:** Appends a CodeChunk with `chunk_type="class"`, then creates a nested visitor with `parent=node.name` and calls `nested.generic_visit(node)` to find methods. Extends chunks. This emits BOTH the class-level chunk AND individual method chunks.

**`parse_python(path: Path, source: str) -> list[CodeChunk]`** — public function:
- Calls `ast.parse(source, filename=str(path))`
- Creates `_ChunkVisitor(source_lines, str(path))` and calls `visitor.visit(tree)`
- Returns `visitor.chunks`

Import CodeChunk from `docgen.models`. Use stdlib `ast` and `pathlib.Path` only — no external dependencies.
  </action>
